This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is an FRC (FIRST Robotics Competition) robot project for **Team 4775 (Aztech)**, built on WPILib 2026 with Java 17 and GradleRIO. The robot uses CTRE Phoenix 6 hardware (TalonFX motors, swerve drivetrain) and Limelight cameras for vision.

## Build & Deploy Commands

```bash
# Build the project
./gradlew build

# Deploy to the RoboRIO over the network (robot must be connected)
./gradlew deploy

# Run simulation (opens WPILib simulation GUI)
./gradlew simulatejava

# Run tests
./gradlew test

# Build without deploying
./gradlew assemble
```

Team number is read from `.wpilib/wpilib_preferences.json` (currently **4775**).

## Architecture

### Entry Points
- `Main.java` — JVM entry point, starts the WPILib robot loop
- `Robot.java` — Extends `TimedRobot`; handles lifecycle hooks (`robotInit`, `robotPeriodic`, autonomous/teleop/test phases) and publishes telemetry to SmartDashboard
- `RobotContainer.java` — Instantiates all subsystems, binds controller buttons to commands, provides `getAutonomousCommand()`

### Subsystems (`src/main/java/frc/robot/subsystems/`)
- **`CommandSwerveDrivetrain`** — Extends CTRE's `TunerSwerveDrivetrain` and implements WPILib `Subsystem`. Handles field-centric driving, alliance-relative perspective, SysId characterization, simulation, and vision measurement fusion via Kalman filter.
- **`IntakeSubsystem`** — Controls the arm/wrist with two NEO brushless motors via SparkMax (`intakeMotorLeft` at `Ports.Intake.LEFT_SPARK_ID`=18, `intakeMotorRight` at `Ports.Intake.RIGHT_SPARK_ID`=15; right follows left inverted). Roller is a Kraken X44 TalonFX (`Ports.Intake.ROLLER_ID`=28). Indexer is a TalonFX (`Ports.Indexer.ID`=24). All on the RIO CAN bus.
- **`ShooterSubsystem`** — Controls shooter wheels (`rsMotor`/`lsMotor`) and wrist (`twMotor`/`bwMotor`) via TalonFX. Includes RPM calculation helpers using `InterpolatingDoubleTreeMap` for distance-based shot tuning.
- **`DashboardSubsystem`** — Pushes/pulls PID constants to/from SmartDashboard for live tuning (paths: `SmartDashboard/Swerve/RotationPID/`, `SmartDashboard/Intake/PID/`, `SmartDashboard/Shooter/AnglePID/`).

### Commands (`src/main/java/frc/robot/commands/`)
- **`ATTRotation`** — Auto-rotates the drivetrain to align with an AprilTag using a PID loop on Limelight TX. Runs concurrently with `StartShooter` on right trigger.
- **`AprilTagTracker`** — Full AprilTag tracking (translation + rotation).
- **`StartShooter`** / **`StartIntake`** — Spin mechanisms at a given percent output.
- **`IntakePos`** / **`WristPos`** / **`WristSpeed`** — Positional and speed control for intake/wrist mechanisms.
- **`Indexer`** — Controls the indexer motor.

### Generated Code
- **`generated/TunerConstants.java`** — Auto-generated by CTRE Tuner X. Contains all swerve module hardware IDs, gear ratios, PID gains, and wheel offsets. **Do not hand-edit**; regenerate via Tuner X if drivetrain hardware changes.

### Key Files
- **`Ports.java`** — Centralized hardware IDs: swerve module TalonFX/CANcoder IDs (1–12, CANivore bus), Pigeon2 (ID 13), Shooter TalonFX (20/21), Hood TalonFX (22/23), Indexer TalonFX (24), Intake SparkMax (18/15) and roller TalonFX (28, RIO bus). **All CAN IDs live here, not in Constants.java.**
- **`Constants.java`** — Static constants for the Limelight name (`Constants.Limelights.shooterLimelightName = "limelight"`), AprilTag IDs per alliance, PID defaults, shooter RPM/angle setpoints, and field geometry.
- **`LimelightHelpers.java`** — Third-party vendor utility for the Limelight camera API. Do not modify.
- **`Telemetry.java`** — Logs swerve state to SmartDashboard/SignalLogger via `telemeterize()` callback registered in `RobotContainer`.

### Controller Bindings

Two Xbox controllers are used. See `RobotContainer.configureBindings()` for the authoritative layout.

**Driver 1 — `joystick` (port 0): chassis + intake**
| Input | Action |
|---|---|
| Left stick | Field-centric translation |
| Right stick X | Rotation |
| Right trigger (hold) | Run intake rollers forward (absorb, 75%) |
| Left trigger (hold) | Run intake rollers reverse (expel, 75%) |
| Right bumper | Intake arm down (position 4, collect) |
| Left bumper | Intake arm up (position 0, home) |
| X (hold) | Indexer forward (50%) |
| B (hold) | Indexer reverse (50%) |
| Start | Reset field-centric heading |
| Back + Y / Back + X | SysId dynamic forward/reverse |
| Start + Y / Start + X | SysId quasistatic forward/reverse |

**Driver 2 — `joystick2` (port 1): shooter**
| Input | Action |
|---|---|
| Right trigger (hold) | ShootSequence with auto-aim to AprilTag |
| Left trigger (hold) | ShootSequence manual (no auto-aim) |
| Right bumper | Toggle auto-aim calculation |

### Vision
One Limelight 2 Plus: `"limelight"` (shooter-facing, pipeline 1 for AprilTags), configured via `Constants.Limelights.shooterLimelightName`. `Robot.java` publishes `TargetDistance` and `TargetTX` to SmartDashboard each periodic cycle. Alliance-specific AprilTag IDs are defined in `Constants.AprilTags.RedAlliance` and `BlueAlliance`.

### Hub Activity Logic
`isHubActive()` (in both `Robot` and `RobotContainer`) determines hub status from match time and game-specific message data. The game message char `'R'`/`'B'` encodes which alliance won autonomous, affecting Shift 1–4 hub availability windows (130s, 105s, 80s, 55s, 30s thresholds).
