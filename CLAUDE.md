## Project Overview

This is an FRC (FIRST Robotics Competition) robot project for **Team 4775 (Aztech)**, built on WPILib 2026 with Java 17 and GradleRIO. The robot uses CTRE Phoenix 6 hardware (TalonFX motors, swerve drivetrain) and Limelight cameras for vision.

## Build & Deploy Commands

```bash
# Build the project
./gradlew build

# Deploy to the RoboRIO over the network (robot must be connected)
./gradlew deploy

# Run simulation (opens WPILib simulation GUI)
./gradlew simulatejava

# Run tests
./gradlew test

# Build without deploying
./gradlew assemble
```

Team number is read from `.wpilib/wpilib_preferences.json` (currently **4775**).

## Architecture

### Entry Points
- `Main.java` ŌĆö JVM entry point, starts the WPILib robot loop
- `Robot.java` ŌĆö Extends `TimedRobot`; handles lifecycle hooks (`robotInit`, `robotPeriodic`, autonomous/teleop/test phases) and publishes telemetry to SmartDashboard
- `RobotContainer.java` ŌĆö Instantiates all subsystems, binds controller buttons to commands, provides `getAutonomousCommand()`

### Subsystems (`src/main/java/frc/robot/subsystems/`)
- **`CommandSwerveDrivetrain`** ŌĆö Extends CTRE's `TunerSwerveDrivetrain` and implements WPILib `Subsystem`. Handles field-centric driving, alliance-relative perspective, SysId characterization, simulation, and vision measurement fusion via Kalman filter.
- **`IntakeSubsystem`** ŌĆö Controls `intakeMotor` (CAN ID 3) and `indexerMotor` (CAN ID 4) via TalonFX.
- **`ShooterSubsystem`** ŌĆö Controls shooter wheels (`rsMotor`/`lsMotor`) and wrist (`twMotor`/`bwMotor`) via TalonFX. Includes RPM calculation helpers using `InterpolatingDoubleTreeMap` for distance-based shot tuning.
- **`DashboardSubsystem`** ŌĆö Pushes/pulls PID constants to/from SmartDashboard for live tuning (paths: `SmartDashboard/Swerve/RotationPID/`, `SmartDashboard/Intake/PID/`, `SmartDashboard/Shooter/AnglePID/`).

### Commands (`src/main/java/frc/robot/commands/`)
- **`ATTRotation`** ŌĆö Auto-rotates the drivetrain to align with an AprilTag using a PID loop on Limelight TX. Runs concurrently with `StartShooter` on right trigger.
- **`AprilTagTracker`** ŌĆö Full AprilTag tracking (translation + rotation).
- **`StartShooter`** / **`StartIntake`** ŌĆö Spin mechanisms at a given percent output.
- **`IntakePos`** / **`WristPos`** / **`WristSpeed`** ŌĆö Positional and speed control for intake/wrist mechanisms.
- **`Indexer`** ŌĆö Controls the indexer motor.

### Generated Code
- **`generated/TunerConstants.java`** ŌĆö Auto-generated by CTRE Tuner X. Contains all swerve module hardware IDs, gear ratios, PID gains, and wheel offsets. **Do not hand-edit**; regenerate via Tuner X if drivetrain hardware changes.

### Key Files
- **`Constants.java`** ŌĆö All static constants: CAN IDs, Limelight names (`"limelight"`, `"limelight2"`), AprilTag IDs per alliance, PID defaults, field geometry. Note: many CAN IDs are placeholder `0` and must be updated for hardware.
- **`LimelightHelpers.java`** ŌĆö Third-party vendor utility for the Limelight camera API. Do not modify.
- **`Telemetry.java`** ŌĆö Logs swerve state to SmartDashboard/SignalLogger via `telemeterize()` callback registered in `RobotContainer`.

### Controller Bindings (Xbox Controller port 0)
| Input | Action |
|---|---|
| Left stick | Field-centric translation |
| Right stick X | Rotation |
| Right trigger (hold) | Start shooter + auto-rotate to AprilTag |
| Left trigger (hold) | Run intake |
| Right bumper | Toggle auto-aim calculation |
| Left bumper / Start | Reset field-centric heading |
| Y / X | Intake position up/down |
| B / A | Wrist speed up/down |

### Vision
Two Limelights: `"limelight"` (shooter-facing, pipeline 1 for AprilTags) and `"limelight2"` (extra). `Robot.java` publishes `TargetDistance` and `TargetTX` to SmartDashboard each periodic cycle. Alliance-specific AprilTag IDs are defined in `Constants.AprilTags.RedAlliance` and `BlueAlliance`.

### Hub Activity Logic
`isHubActive()` (in both `Robot` and `RobotContainer`) determines hub status from match time and game-specific message data. The game message char `'R'`/`'B'` encodes which alliance won autonomous, affecting Shift 1ŌĆō4 hub availability windows (130s, 105s, 80s, 55s, 30s thresholds).
